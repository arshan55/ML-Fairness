# End-to-End Fairness Lifecycle in Machine Learning

## 1. Project Overview
This repository contains a comprehensive pipeline for evaluating and mitigating algorithmic bias in Machine Learning models. Moving beyond simple post-hoc audits, this framework establishes a **full-lifecycle approach** that features early dataset bias symptom detection and PyTorch-based differentiable mitigation strategies. It is explicitly designed to identify and resolve demographic fairness trade-offs in clinical and financial tabular datasets.

## 3. Data Sources & Preprocessing
The project dynamically targets the **UCI Machine Learning Repository** to pull industry-standard benchmark datasets critical to proving ML Fairness:
- **Adult Income Dataset (US Census)**: Predicts income > $50k. Historically biased against gender (`sex`) and race.
- **Heart Disease Dataset**: Focuses on clinical correlations heavily prone to sampling bias.

**Preprocessing Strategy (`01_preprocessing`):** 
Raw attributes are passed through `LabelEncoder` and `StandardScaler` to handle multi-scalar anomalies. Before models are trained, **Bias Symptoms** are quantified strictly using statistical analysis:
1. **Mutual Information**: Evaluates correlation between independent variables and protected attributes.
2. **Earth Mover's Distance (Wasserstein)**: Measures cross-distribution skew on the ground truth variables. Both metrics act as 'early warning' bias diagnostic indicators.

## 4. Model Architectures & Pipeline Strategy

### A. The Baselines (`02_baseline_training`)
Standard **Random Forest** and **Logistic Regression** models are trained to build the absolute limits of accuracy on the datasets. Because they are not fairness-constrained, they maximize `AUC-ROC` but fail Demographic Parity (DP) audits.

### B. Fairness Auditing Framework (`03_fairness_evaluation`)
Uses native evaluations from `fairlearn`:
- **Demographic Parity Difference**
- **Equalized Odds Difference**
- **Disparate Impact**
*Bonus:* Includes a simulated **Propensity Score Matching (Matched Counterpart)** audit mapping via K-Nearest Neighbors to isolate socioeconomic variables from explicitly discriminatory protected traits.

### C. Mitigation (`04_mitigation`)
Addresses "in-processing" resolution. A PyTorch-based Fair Neural Network is developed with a custom **Log-Loss function combined with a Differential Demographic Parity Regularizer**. By tracking the squared variance of the mean prediction outcomes between Group 0 (disadvantaged) and Group 1 (advantaged), the network's weights are penalized for discriminatory gradients. 

## 5. Outputs and Generation for Publications
Rather than a standard web app, this pipeline optimizes specifically for **academic research exportation**: 

1. **`05_paper_metrics_exporter`**: Combines data into a `csv` that rigorously logs the "Accuracy-Fairness Trade-off" (e.g. tracking how much Demographic bias was reduced over exact percentile drops in accuracy).
2. **`06_journal_visualizations`**: Emits `seaborn` plotting parameters to generate publication-ready PNG figures matching the Journal formats:
   - *Figure 2*: Bar Chart comparing Baseline Accuracies.
   - *Figure 3*: ROC Curve mapping the integrity of the Fair NN vs the DNN/Random Forests.
   - *Figure 4*: Confusion Matrix heatmap for diagnostic transparency.
   - *Figure 6*: Violin Plots comparing prediction score distributions across different demographic attributes.

## 6. Installation & Quickstart Execution

To run the pipeline locally, ensure Python 3.10+ is installed, then execute:

```bash
# 1. Install required packages
pip install -r requirements.txt

# 2. Automatically run all six phases of the Project Pipeline
python run_pipeline.py
```

### Alternatively (Jupyter Interface)
For a visual, cell-by-cell breakdown of the framework, we provide an interactive compiler:
```bash
python make_notebook.py
```
Open `Fairness_Pipeline.ipynb` in your preferred editor.
